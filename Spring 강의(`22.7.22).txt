1. JPA 가 무엇인가?
   1) 자바 명령어를 SQL로 잘 번역해주는 역할을 하는 문법
   2) 스프링을 위해 태어난 녀석
   3) Java로 코드를 작성하면 SQL로 번역해줄 뿐만 아니라, 기본적인 기능이 거의 완벽하게 들어가있다.

2. Controller, Service, Repository 가 무엇인가요?
   1) Controller -> 자동읍답기
    - 클라이언트의 요청(Request)을 전달받는 코드를 Controller 라고 부릅니다.
        → JSON 만을 돌려주는 것은 RestController 라고 부릅니다. 
    - 우리는 이런 역할을 하는 빵틀을 새롭게 만들면 되는 것이군요!

   2) Repository
    - class의 한 역할
    - 데이터를 접근 할 때 사용하는 하나의 도구
    - JPA를 작동시키는 매개체

3. restAPI의 put 과 patch 는 어떤 차이점이 있을까요? 어떤 경우에 사용하면 좋을까요?

4. 우리가 브라우저에서 구매한 도메인 주소를 입력하면 만들어 놓은 aws EC2 서버까지 전달이 되어서 해당 서버에서 요청을 보낼 수 있습니다.
   이 과정이 상세하게 어떻게 진행되는지 그림으로 한 번 그려볼까요?

5. Java를 활용하여 필요한 클래스를 구상 할 수 있어요.

6. Lombok과 JPA를 이용하여 원하는 데이터베이스를 만들고 활용 할 수 있어요.
   Lombok : 자동생성해줌으로써 코드를 절약 할 수 있도록 도와주는 라이브러리

7. Spring Boot를 기반으로 CRUD(Create, Read, Update, Delete) 기능이 포함된 REST API를 만들 수 있어요

8. AWS에 Spring Boot 서비를 배포 할 수 있어요
